Mere file upload

Multipart/form-data POST parsed on the Mere side by contrib/http/multipart.mere. Uploaded files are stored in a log-structured KV and served back via GET /files/<id> with the right Content-Type.

Text files only for this demo (.txt, .md, .json, .csv, .html, .log). Binary formats break the UTF-8 round-trip through Mere strings — a base64 wrapper would fix it but isn't wired up yet.
